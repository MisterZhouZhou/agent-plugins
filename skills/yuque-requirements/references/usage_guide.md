# 语雀前端需求提炼 Skill 使用说明

## 这是什么

这个 Skill 用来把语雀文档自动整理成两份文件：

1. 原始清洗文档：把语雀内容去掉多余 HTML、样式和无关包裹后，输出为 Markdown。
2. 前端技术文档：只提炼前端开发需要关注的内容，按固定模板输出需求文档。

适合的场景：

- 产品或运营给了一篇语雀 PRD，希望快速整理成前端可实现的技术文档
- 语雀文档里混有图片、HTML 标签、样式内容，阅读和提炼成本高
- 需要保留原始需求标题、接口信息、状态枚举、字段映射、参考代码路径

## 使用前准备

使用前请准备以下信息：

1. 语雀文档链接
2. 语雀 Token
3. 如果有的话，再补充参考上下文

语雀 Token 要求：

- 在语雀个人设置里创建
- 至少具备读取权限
- 如果文档是私有知识库，Token 对应账号必须有该文档访问权限

## 最简单的用法

直接在对话里提供语雀链接和 Token。

示例：

```text
请使用 yuque-frontend-requirements skill 处理这个语雀文档：
链接：https://www.yuque.com/owner/repo/doc-slug
token：你的语雀 Token
```

如果你还希望输出内容更贴近现有前端项目，可以一起补充参考上下文。

示例：

```text
请使用 yuque-frontend-requirements skill 处理这个语雀文档：
链接：https://www.yuque.com/owner/repo/doc-slug
token：你的语雀 Token

参考上下文：
- 日期选择组件参考：xxx/FormSearch.tsx 中的 RangePicker
- 查询按钮参考：xxx/AuditSearch.tsx
- 列表请求参考：xxx/index.tsx 中 updateTable 的写法
```

## 推荐输入格式

为了让输出更稳定，建议按下面格式提供信息：

```markdown
# 参考上下文
- 参考代码路径1
- 参考代码路径2

语雀链接：https://www.yuque.com/owner/repo/doc-slug
token：你的语雀 Token
```

如果没有参考上下文，可以只给语雀链接和 Token。

## 处理后会生成什么

默认会生成两份 Markdown 文件：

1. `{语雀标题}-原文档.md`
2. `{语雀标题}-技术文档.md`

通常会放在项目的 `outputs/` 目录下。

例如：

- `outputs/增加检测维保长图展示入口-原文档.md`
- `outputs/增加检测维保长图展示入口-技术文档.md`

## 技术文档会怎么写

生成的技术文档会遵循以下规则：

1. `# 参考上下文`
   如果你没有提供额外上下文，这里会写 `暂无`。

2. `# 功能需求`
   会尽量保留原始需求标题，并改写成前端可实现描述。

3. `## 接口描述`
   如果语雀文档没有接口信息，会明确写 `暂无`，不会擅自补猜。

4. `## 工作流程`
   会固定保留“先理解需求，再拆分任务，再开发”的协作约束。

## 适合补充给 Skill 的信息

如果你希望输出更像团队内部的前端技术文档，建议补充这些信息：

- 参考代码路径
- 参考组件写法
- 字典获取方式
- 列表请求示例
- 详情接口示例
- 页面跳转参考
- 表格组件参考

这些内容会被写入 `# 参考上下文`，方便后续开发继续沿用已有实现风格。

## 常见问题

### 1. 返回 404

一般有这几种原因：

- Token 没有私有知识库访问权限
- `repo namespace` 或 `doc slug` 不正确
- 请求没有带 `raw=1`
- 请求没有带 `User-Agent`

### 2. 生成的技术文档里很多地方是“暂无”

这通常不是 Skill 出错，而是原文档没有提供足够信息。常见缺失项包括：

- 接口名称
- 请求路径
- 请求参数
- 响应字段
- 状态枚举
- 字段映射

这种情况下，Skill 会明确标记 `暂无` 或 `待确认`，不会擅自猜测。

### 3. 为什么“参考上下文”里没有语雀文档摘要

这是故意设计的。`# 参考上下文` 只承接你额外提供的实现锚点，比如参考代码路径、组件写法、接口调用方式。

### 4. 为什么有些条目下面会出现短横线列表

当一个编号点下有多种状态、类型或分支逻辑时，会优先用短横线列表展开，方便阅读和后续开发。

## 一份完整示例

```markdown
# 参考上下文
- 日期选择组件：参考 sample-app/src/pages/xxx/FormSearch.tsx 中的 RangePicker
- 查询按钮：参考 sample-app/src/pages/xxx/AuditSearch.tsx
- 列表请求：参考 sample-app/src/pages/xxx/index.tsx 中 dispatch.xxx.updateTable

语雀链接：https://www.yuque.com/owner/repo/doc-slug
token：你的语雀 Token
```

## 给使用者的建议

1. 先确保自己能在浏览器中打开对应语雀文档。
2. 私有文档一定要确认 Token 权限。
3. 如果希望生成结果更贴近团队代码风格，尽量补充参考上下文。
4. 对生成结果里的 `待确认` 和 `暂无`，最好回头补齐产品或后端信息。
