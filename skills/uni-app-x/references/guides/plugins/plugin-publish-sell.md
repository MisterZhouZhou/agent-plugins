# 插件发布与销售变现

## 为什么发布插件

- 通用功能模块可发布到 DCloud 插件市场供其他开发者复用。
- 插件市场支持赞赏、广告解锁、计费销售、评价等机制。
- 插件下载量、评分和排行榜可为作者带来曝光和信誉。
- 插件作者要获取收益，需要在插件市场实名注册并填写财务信息。

## 插件制作注意

- 遵循插件目录规范，尤其要确定插件 ID。
- 插件包不应包含 `unpackage` 目录。
- 插件包最好不包含 `node_modules` 目录。
- 插件包不应包含 `.git`、`.svn` 等版本控制目录。
- 前端插件建议使用 scss 预处理并引用 `uni.scss` 变量，方便不同插件组合时保持风格一致。

插件 ID 命名规范：

- 格式为 `作者ID-插件英文名称`，如 `xiaoming-tag`。
- 作者 ID 和插件名称只能包含英文、数字。
- 作者 ID 不能使用 `DCloud`、`uni` 等关键字，长度至少 2 位。
- 插件英文名称应直观表达插件作用，如 `eshop`、`button`。

## 推荐发布方式

- HBuilderX 3.1 起支持 `uni_modules`，建议插件作者优先用 `uni_modules` 开发和发布插件。
- 推荐先在自己的项目下创建 `uni_modules`，本地开发调试无误后，对 `uni_modules/<plugin_id>` 目录右键发布到插件市场。
- 发布新版本时仍通过插件目录右键「发布到插件市场」。
- 插件基本信息如中文名称、描述、关键词、`readme.md` 等，可发布后通过「修改插件基本信息」调整。

## 插件类型与发布要点

### 前端组件

- 前端组件指 Vue / nvue / wxml 等技术开发的页面内嵌组件。
- 前端组件细分为 Vue 组件、UVue 组件、nvue 组件、小程序自定义组件等。
- 该分类不包含 UTS 原生组件。
- 小程序组件的一级目录通常从 `components` 改为 `wxcomponents`。
- 组件不能包含项目根目录级的 `manifest.json`、`pages.json`、`App.vue`、`main.js` 等文件。

### 付费前端组件

- HBuilderX 4.19 起，DCloud 通过云编译支持前端付费插件的源码保护。
- 插件作者设置加密后，使用者下载到的是加密版本。
- 运行或发行时，uni-app x 编译器会将加密插件发送到 DCloud 云编译服务器编译，再把结果混入本地编译工程。
- 云编译和解密只涉及加密插件，不会上传使用者项目源码整体。
- 前端组件付费插件目前仅支持 uni-app x 下 `uni_modules` 形式、符合 easycom 规范的组件。
- 前端组件付费插件可在 Web、App 等全平台加密。
- 付费前端组件会自动加密所有 `vue`、`uvue`、`uts`、`js`、`ts` 文件，但 `pages`、`hybrid`、`static` 目录除外。
- 付费前端组件不支持混搭 `utssdk` 目录，不支持页面加密。
- 插件需要对外导出方法或类型时，应在插件根目录 `index.uts` 中导出；使用者不能直接引入插件内部文件。
- 前端组件付费插件云端独立编译，不能引入项目内其他目录资源；需要使用者数据时，应通过 API / props 传入。
- App Android 平台不支持导出 Vue plugin 和 mixin。
- 首次运行或发行到指定平台时会触发云端编译，需联网且 HBuilderX 登录账号具备购买、试用或协作权限；后续可使用本地缓存。

### 页面模板与项目模板

- 页面模板如果需要导入时注册到使用者项目的 `pages.json`，可提供 `pages_init.json`。
- Vue 页面模板和 nvue 页面模板目录结构要求相同。
- 页面模板打包时需要包含 `manifest.json`、`pages.json` 等文件。
- 项目模板打包时不需要包含 `unpackage` 目录。
- 项目模板的 `manifest.json` 不允许包含 DCloud appid 或微信等三方 appid。
- 如果模板包含 uniCloud 云函数目录，应按云端一体项目模板分类处理；不使用云函数时应删除相关目录。

### JS SDK

- JS SDK 目录结构无特殊要求，通常将 JS 文件命名为插件 ID。
- JS SDK 不能包含项目根目录的 `manifest.json`、`pages.json`、`App.vue`、`main.js` 等文件。
- 非 `uni_modules` 的 JS SDK 下载后会进入使用者项目根目录的 `js_sdk` 目录。

### UTS 插件

- UTS 插件发布和开发详见 UTS 插件开发文档。
- UTS 付费插件支持普通授权版和源码授权版。
- DCloud 插件市场会对付费 UTS 插件自动加密：
  - 加密除 `interface.uts` 之外的所有 UTS 文件。
  - 加密 `utssdk/app-android`、`utssdk/app-ios` 下的 Java、Kotlin、Swift 等混编文件。
- UTS 插件试用时，试用者无法查看加密源码，只能用于打包自定义基座，不能正式发布。
- 购买普通授权版后仍无法查看加密源码，提交云打包时由云端验证并解密打包。
- 购买源码授权版后，使用者才能下载插件源码。
- 普通授权版和源码授权版都绑定唯一 appid 和包名；更换任一信息通常需要重新购买授权。
- UTS 加密插件只支持云端传统打包，不支持离线打包，也不支持安心打包。
- UTS 加密插件打包最低需要 HBuilderX 3.7.2+。
- 一般提交 UTS 插件建议只包含 UTS 源码；三方 SDK 如 jar / aar 等建议配置为仓储依赖。
- HBuilderX 4.81 起，UTS 付费插件支持 HarmonyOS 平台。
- 如果发行平台为 Web、小程序或 HarmonyOS，UTS 加密插件需要提交云端编译，目前仅支持 uni-app x 项目。

## 付费插件销售

- 插件市场支持对 uniCloud 插件、App 的 UTS / 原生插件、前端组件插件设置付费销售。
- uniCloud 插件、UTS 插件、前端组件插件支持普通授权版、源码授权版。
- App 原生插件仅支持普通授权版，不提供源码。
- 其他类型插件不支持计费销售。
- 未设置价格时默认为免费插件。
- 如果要设置源码授权版价格，必须先设置普通授权版价格。

## 版权保护与绑定

- 普通前端插件因源码开放问题，无法支持计费。
- uniCloud 插件的云函数支持源码保护，未购买源码版的用户无法获取源码。
- 云端一体插件的前端 JS 可加密混淆，但前端代码运行在设备上，仍有被破解风险。
- uniCloud 插件销售与服务空间 `spaceId` 绑定；未经授权的服务空间无法上传加密云函数。
- UTS / App 原生插件、前端组件插件需要使用 DCloud 云打包后才能运行，包括自定义基座；本地打包不支持插件市场这类付费插件。
- UTS / 原生插件、前端组件插件销售与客户端 appid、包名绑定；未经授权的 appid 和包名无法运行付费插件。

## 试用机制

- DCloud 为销售插件提供试用机制，允许先试用后付费。
- uniCloud 插件提供 7 天试用，试用到期后会自动销毁加密云函数。
- UTS / App 原生插件及前端组件插件仅支持在 App 自定义运行基座上试用。
- 试用基座无法脱离 HBuilderX 独立安装，且每次启动会弹出测试 toast。

## 收益与结算

- 用户购买插件后，作者可在插件市场销售订单列表查看订单流水。
- 插件收益包括用户付费收益（销售、赞赏）和广告收益。
- 用户付费收益通常每天凌晨统计前一天数据。
- 广告预估收益通常当天下午 2 点后统计前一天数据，仅作参考，实际以月度账单为准。
- 插件市场每月 1 日出上月月度账单，包含上月用户付费收益和上上月广告收益。
- 待提现账单累计金额达到 100 元以上时可提现，也可合并多笔账单提现。
- DCloud 会收取付费插件和赞赏账单金额的 15% 作为服务费。
- 插件提现有效期为 24 个月，需及时关注。
- 企业认证账号申请提现后需提供发票，电子发票仅支持 PDF 格式。
