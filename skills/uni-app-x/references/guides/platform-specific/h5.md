# H5 and Web Platform

Web builds can use browser capabilities, but Web-only code must not leak into App native builds.

## manifest router

```json
{
  "h5": {
    "router": {
      "mode": "hash",
      "base": "/"
    },
    "devServer": {
      "https": false,
      "port": 8080
    }
  }
}
```

Use `history` mode only when the deployment server rewrites all routes to the Web entry.

## Browser-only code

```uts
// #ifdef WEB
const userAgent = window.navigator.userAgent
console.log(userAgent)
// #endif
```

## Web-only responsive CSS

```vue
<style>
.container {
  width: 750rpx;
}

/* #ifdef WEB */
@media (min-width: 768px) {
  .container {
    width: 720px;
    margin-left: auto;
    margin-right: auto;
  }
}
/* #endif */
</style>
```

## Checklist

- Browser globals are guarded with `WEB`.
- Router mode matches deployment.
- Web-only CSS is inside conditional blocks when not App-compatible.
- App-safe UCSS remains the default for shared styles.
- Test in target browsers, not only HBuilderX preview.
