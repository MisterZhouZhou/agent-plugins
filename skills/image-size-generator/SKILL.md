# Image Size Generator

此技能用于根据用户要求生成不同尺寸的图片资源，适用于图标、封面、社交分享图、截图占位图和其他需要固定宽高输出的静态图片。

## 工作流 (Workflow)

当用户要求生成特定尺寸的图片时，请遵循以下步骤：

1. **确认输出要求**:
   - 提取用户指定的尺寸、数量、命名方式和格式。
   - 如果用户没有指定格式，默认导出 PNG。
   - 如果用户给的是一组尺寸，按原样逐个输出，例如 `16x16`、`48x48`、`128x128` 或 `1200x630`、`800x450`。
   - 默认将输出文件写入项目根目录下的 `assets/`；如果目录不存在，先创建它。
   - 默认不保留生成过程中使用的源文件；只有用户明确要求保留源文件时，才保留 SVG 或中间高分辨率素材。

2. **准备源图像**:
   - 如果用户提供了现有的 SVG 或高分辨率 PNG，请优先使用它作为源文件。
   - 如果用户只描述了一个简单图形、图标、徽标或几何风格画面，请直接生成一个干净的 SVG 源文件，便于后续无损缩放。
   - 如果目标尺寸存在多种宽高比，不要把同一个源图直接拉伸到所有尺寸。应先为对应宽高比创建匹配画布的 SVG，再导出 PNG，避免变形。
   - 如果源图是为了本次导出临时创建的，在完成导出和校验后默认删除；仅在用户明确要求保留源文件时保留。

3. **生成所需尺寸**:
   - 使用 macOS 的 `sips` 命令行工具进行格式转换和尺寸导出。
   - `sips -z` 的参数顺序为 `<height> <width>`，输出命名建议包含尺寸信息。
   - 输出路径默认使用 `assets/`，例如 `assets/icon-128.png`、`assets/hero-1200x630.png`。
   - 示例：将 `hero.svg` 导出为多个尺寸的 PNG
     ```bash
     mkdir -p assets
     sips -s format png hero.svg --out assets/hero-1200x630.png -z 630 1200
     sips -s format png hero.svg --out assets/hero-800x450.png -z 450 800
     sips -s format png hero.svg --out assets/hero-512x512.png -z 512 512
     ```
   - 示例：批量导出一组正方形尺寸
     ```bash
     mkdir -p assets
     for size in 16 32 48 128 256; do
       sips -s format png icon.svg --out "assets/icon-${size}.png" -z "$size" "$size"
     done
     ```

4. **验证输出结果**:
   - 使用 `ls` 确认文件已经生成。
   - 使用 `sips -g pixelWidth -g pixelHeight <file>` 检查最终尺寸是否与要求一致。
   - 如果用户要求多个比例，逐个核对关键文件，确保没有拉伸、裁切或透明背景丢失的问题。
   - 校验完成后，如果用户没有要求保留源文件，删除本次临时创建的 SVG 或中间文件，只保留最终交付图片。

## 注意事项 (Important Notes)

- 优先使用 SVG 作为源文件，这样更适合导出多个尺寸。
- 当目标是固定尺寸的矩形图时，先让源文件画布与目标宽高比一致，再使用 `sips` 导出。
- `sips` 适合尺寸转换和格式导出，不适合复杂图像编辑；这个技能重点是“生成/整理源图并导出指定尺寸”，不是做高级修图。
- 如果用户要求的是平台规范图（例如 OG 图、Banner、缩略图），文件名应直接反映用途和尺寸，便于后续接入项目。
- 默认交付物应尽量精简，除非用户明确要求，否则不要保留源 SVG、临时 PNG 或测试文件。
