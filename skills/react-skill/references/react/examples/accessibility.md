# Accessibility

## Use for

Keyboard behavior, semantic HTML, focus management, forms, dialogs and ARIA.

## Guidance

- Use native elements first: `button`, `a`, `label`, `fieldset`, `nav`, `main`.
- Every interactive control needs a visible or programmatic accessible name.
- Manage focus when opening modals, drawers and route-level workflows.
- Use ARIA to fill semantic gaps, not to disguise non-semantic markup.

## Checks

- Can the workflow be completed with keyboard only?
- Do form errors connect to the relevant fields?
- Are loading and error states announced where needed?
