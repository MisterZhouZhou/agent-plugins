# ESLint

## Use for

React lint rules, Hooks checks, formatting boundaries and migration to flat config.

## Guidance

- Keep `eslint-plugin-react-hooks` enabled for Hook correctness.
- Do not disable exhaustive deps broadly.
- Follow existing formatter integration, whether Prettier is separate or inside lint.

## Checks

- Are test files using the right globals?
- Are generated or build output folders ignored?
